The Reading

Both papers wrestle with the same theme ENGRAM lives in: how a system remembers the right things over time — and how to know when its memory is good.

ALMA — Meta-learning Memory Designs

Learning to continually learn: a Meta Agent searches over memory designs expressed as executable code, evaluates them on real agentic tasks, and archives the winners. The learned designs beat hand-crafted memory across four domains — and cost less.

01

The ENGRAM challenge

The Kit

ENGRAM is a small challenge for memory systems in evolving workspaces. Your program receives dated workspace states in chronological order. It may write anything it wants into its own memory directory. When ENGRAM asks a question, your program answers from the current state plus that memory — without hindsight.

The two-command contract

Update
./run.sh update STATE_DIR MEMORY_DIR

Ingest one dated workspace state and fold it into persistent memory under MEMORY_DIR. No future states are ever shown.

Answer
./run.sh answer STATE_DIR MEMORY_DIR QUESTION_JSON ANSWER_JSON

Answer from the current state plus prior memory, and cite evidence paths. No answer keys, no peeking ahead.

Four kinds of fact your memory must tell apart

Current

True as of the latest state — the answer you should usually give.

Stale

Was true once, may no longer be. Don't mistake it for current.

Superseded

Explicitly replaced by a newer fact. Track what overrode what.

Uncertain

Not enough evidence yet. Say so rather than guessing.

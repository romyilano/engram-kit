Page 02

Purpose:
The problem with old benchmarks — they test dialogue/chit-chat, but real agent memory is a trajectory of states, actions, observations, and tool outputs: machine-generated (JSON, ASCII tables, code), causally linked, and objective (no fluff).

Audience note:
Builds on page 1. Contrast two kinds of "memory": a chatty conversation vs. a machine work-log.

Panel Count:
6

Image Prompt:

Create a six-panel educational manga page built around a clear LEFT-vs-RIGHT contrast (dialogue world vs. trajectory world).

Style:
Soft-color educational manga.
Light watercolor / gentle anime palette — warm cream paper, soft ink linework, muted pastel washes.
One restrained accent: recall-green (#00e0a4), used only for memory, recall, and "it worked" moments.
Clean confident linework, generous negative space, friendly but precise.
Calm, contemplative, explanatory tone — diagrams integrated naturally into the scene.
Not gritty. Not neon. Not chibi. Not glossy 3D. Not photorealistic.

Characters in this page:
- ENGY: a small, rounded librarian-robot with a soft matte-cream chassis, one big recall-green camera-lens eye, stubby arms, an open NOTEBOOK in its chest (its memory), and a small satchel.
- SENSEI: a calm middle-aged human archivist with round glasses, a soft cardigan, neat greying hair, holding a slim wooden pointer.
- THE LONG ROAD / TRAJECTORY: a road strung with machine-generated scrolls — little JSON tags, small ASCII tables, code snippets — each a state, action, observation, or tool output, linked nose-to-tail by a faint recall-green causal thread.

Setting:
A split scene: on one side a warm chatty café (speech bubbles), on the other the machine-scroll road.

Panel 1 (left side):
A cozy café full of floating speech bubbles and small talk — "how are you?", weather chit-chat. Label it as "old benchmarks: dialogue." Pleasant but fluffy.

Panel 2 (right side):
The trajectory road: terse machine scrolls pinned along it — a JSON tag, an ASCII table, a code snippet — each an action/observation/tool-output. No chit-chat. Label "real agent memory: trajectory."

Panel 3 (close, three representation types):
Detail of three scroll types side by side, integrated as a little exhibit: { } JSON, an ASCII grid table, and a `code` block. Caption point: many machine representations, not free-form prose.

Panel 4 (medium, causality):
Sensei's pointer traces the recall-green thread linking one scroll to the next — an action causes a state change that constrains the next observation. Cause and effect, in order.

Panel 5 (close, objectivity):
A scroll being weighed: it's dense and objective (facts, values) with the café's chit-chat bubbles popping away as irrelevant. Signal, not noise.

Panel 6 (two-shot, the thesis):
Engy and Sensei standing on the trajectory road, the café fading behind. The point lands: test memory on trajectories, not conversations. Dialogue space.

Camera directions:
Hold the left/right contrast clearly in panels 1–2; make the scroll types legible in panel 3; use recall-green only on the causal thread; end on the road.

Lettering:
Leave room for English dialogue and captions.
- Panel 1 — (caption) "Old memory tests look like *conversations*."
- Panel 2 — (caption) "But an agent's real memory is a *trajectory* of states, actions, observations, tool outputs."
- Panel 3 — Sensei: "JSON, tables, code — machine-generated, not free-form chatter."
- Panel 4 — Sensei: "And it's *causal*: each action changes the world, which shapes what comes next."
- Panel 5 — Engy: "Dense and objective — no small talk to wade through."
- Panel 6 — Sensei: "So a fair test must use trajectories. That's what AMA-Bench does."

Output:
A single finished manga page.
